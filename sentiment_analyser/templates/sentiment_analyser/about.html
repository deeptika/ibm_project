Twitter Sentiment Analysis

A predictive model for analyzing the sentiments of tweets. Has an accuracy of 95%

To access the project files, click here.

Project Files


Purpose Of This Project

The sentiment analysis of Indians after the "extension of lockdown" announcements are to be analyzed by considering the relevant hashtags on Twitter and building a predictive model to understand the behavior of people in the event of further extension of the lockdown, along with a dashboard for the visualization of the results of prediction.

Tools Used

Google Colaboratory, IBM cloud, Django for deployment, Python 3, pandas, numpy, keras, nltk, Twitter, matplotlib, VADER

MODEL ADOPTED

After training and testing several models the model we decided upon Logistic Regression with Term Frequency(TF) and Inverse Document Frequency(IDF) (i.e) LR with TF-IDF is built with an accuracy of 95%



Methodology

1) The dataset containing only tweet id is scraped , and tweets are extracted using hydrator app
2) Once the tweets are obtained , it is pre-processed, that involves removal of :
- html tags
- special characters
- punctuation marks
- converting to lower case characters
- replacing emoji
- emoticons
- short words
- tokenizing
- removing stop words
- stemming
3)The data is then split into training and test set , for validation
4) The data is trained with logistic regression classifier
5) Test data is used to predict the sentiments
6) The Model is then evaluated.

Stemming is the process of reducing inflected (or sometimes derived) words to their word stem, base or root form—generally a written word form. The stem need not be identical to the morphological root of the word; it is usually sufficient that related words map to the same stem, even if this stem is not in itself a valid root. Many search engines treat words with the same stem as synonyms as a kind of query expansion, a process called conflation.
